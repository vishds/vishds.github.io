Odin's defer differs from Go's defer

As per the Go Blog:

A defer statement pushes a function call onto a list. The list of saved calls is executed after the surrounding function returns.

Odin's defer implementation is what we (atleast I) expect to happen. Odin's Overview states:

A defer statement defers the execution of a statement until the end of the scope it is in.

Media Transfer Protocol

Devices such as android phones connect with linux via MTP. lsusb will recognise these devices, but lsblk will not recognise them as blocks (storage device). Additional packages like mtptools are required to transfer data to such devices.
